The Wall of Algorithms

Next we are going to use a technique similar to the song In C. It's performed with a group of musicians around the size of 35 where everyone has the same sheet music with the same number of bars but each musician decides how long to play a given bar for. The result is a lot of voices playing a coherent piece, but if you took any two or three voices it might sound strange or off.

Instead of musicians we will use our generated voices at diffent base pitches. We'll also have 10 voices instead of 35, two voices centered at 75Hz, three at 300Hz, three at 600Hz and two at 1200Hz. Notice that is nothing connecting the voices, they are entirely indepenedant. But the result is (sometimes) a coherent score. Here's the program minus the synths: 

(
  var basefreq = 400, complexity = 6, voices = [], basefreqs, measurelength, se
  basefreqs = [75, 75, 300, 300, 300, 600, 600, 600, 1200, 1200];
  voices = [];
  seed = 1000000.rand;
  seed.postln;                // Printing the seed so it isn't lost forever.
  thisThread.randSeed = seed;
  measurelength = 4.rand + 1;
  i = 0;
  basefreqs.do({|freq|
    // Create one voice for each of the frequencies in |basefreqs|.
    voices = voices.add(
      Pseed(seed + i, Pbind(
        \numerator, Pwhite(1, complexity),
        \denominator, Pwhite(1, complexity),
        \dur, Pconst(measurelength, 2 ** Pwhite(-3, 1)), // Our duration is a power of two.
        \amp, Pwhite(0.05, 0.2),
        \legato, Pwhite(0.01, 0.3),
        \instrument, \krgn_gen_fmdevil,
        \index, Pwhite(0, 3),
        \freq, Pkey(\numerator) / Pkey(\denominator) * freq,
      ))
    );
    i = i + 1;
  });

  ~players = [];
  voices.do({|voice|
    ~players = ~players.add(voice.play);
  });
)
(
  ~players.do({|player|
    player.stop;
  });
  ~players = [];
)
